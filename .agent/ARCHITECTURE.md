# Architecture

- `src-tauri/src/main.rs` owns Tauri commands, tray/hotkey behavior, onboarding, updater integration, model lifecycle, and the `Idle → Recording → WaitingForModel → Transcribing → Injecting` state machine.
- `src-tauri/src/audio.rs` captures microphone samples only in memory, mixes channels to mono, resamples to 16 kHz PCM 16-bit, tracks level, and surfaces device errors.
- Optional playback ducking uses the Windows Core Audio default render endpoint. A recording-scoped guard stores the original master-volume scalar, sets it to 20% while capturing, and restores it on stop, cancellation, or task failure.
- `src-tauri/src/platform_windows.rs` captures/revalidates foreground process identity; owns per-format clipboard snapshot/restore, Unicode input, work-area/fullscreen queries, sounds, and the Worker Job Object.
- Windows target capture runs before runtime/model checks and only requires a valid foreground window/process. Completion validation uses the stable foreground process ID because Chromium/Electron may omit `hwndFocus` or recreate internal/top-level HWNDs. Text goes to the current cursor in the originally captured application; changing processes falls back to clipboard.
- Clipboard paste is the default output path. Rain deep-copies movable clipboard formats and enhanced metafiles, restores them after a 300 ms paste window only when the sequence still belongs to Rain, and preserves concurrent user copies. GDI bitmap handles are skipped because their `CF_DIB` representation carries the bitmap data.
- `src-tauri/src/models.rs` is the manifest-driven repository layer. It handles resumable official downloads, per-file SHA-256 checks, atomic promotion, directory/ZIP import, custom-model marking, verification, independent HTTPS catalog updates, retained old versions, and root-confined deletion. Schema 2 model definitions carry an `asr|asr_preview|text_polish` purpose plus a runtime repository and CPU/NVIDIA component mapping.
- `src-tauri/src/runtimes.rs` manages the separately published native sherpa-onnx CPU Worker plus Python CPU/NVIDIA Workers. A model download resolves its mapped component, installs any missing runtime before the weights, and reports one combined progress stream. Runtime archives remain resumable, hash-checked, path-confined, and atomic.
- The same runtime repository manages the separate `text-runtimes` root from `src-tauri/resources/text-runtime-manifest.json`. The first text component is the pinned official llama.cpp `b10016` Windows CPU archive; it is not part of the installer.
- `src-tauri/src/text_polish.rs` owns a long-lived localhost-only llama-server with a random port, per-process API key, 30-second load limit, 8-second request limit, Windows Job Object, and idle sleep. The pipeline is `ASR → optional Qwen punctuation/paragraph pass → protected-content validator → injection`; every error or rejected output falls back to the untouched ASR text.
- Text-polish idle time is stored in seconds for backend/runtime compatibility, while the frontend displays and edits the value in minutes and converts at the configuration boundary.
- Runtime lifecycle is reference-based. A standalone removal command returns `RUNTIME_IN_USE` while any installed model maps to that component; deleting a model first stops the Worker to release Windows executable locks, then removes each mapped component only after the last installed model reference disappears. The frontend exposes model deletion only, not separate runtime download/delete controls.
- Every ModelScope HTTP client must send the versioned `RainVibetype/<version>` User-Agent. ModelScope's LFS CDN rejects reqwest's missing/default identity with `403 denied by UA ACL`, even for public models.
- `src-tauri/resources/models.json` is the versioned source of model metadata, official revisions, required files, sizes, hashes, and optimal runtime mappings. SenseVoice maps to sherpa-onnx, Fun-ASR Nano and Paraformer share CPU/NVIDIA FunASR runtimes, and Qwen text cleanup maps to llama.cpp.
- `src-tauri/src/worker.rs` owns the long-lived Worker process protocol: JSON control frames plus binary PCM, request IDs, response timeout, loaded-model caching, unload, crash fuse, and bundled/development executable selection. `AppState` keeps separate final-ASR and streaming-preview `WorkerClient` instances so preview decoding never changes the final SenseVoice result.
- `worker/rain_worker.py` provides one adapter interface for SenseVoice, Fun-ASR Nano, and Paraformer-zh. It loads local paths only and supports CPU/CUDA selection, cancellation semantics, and normalized errors.
- `native-worker/` implements the same line-delimited JSON plus binary PCM contract in Rust with sherpa-onnx. It loads managed SenseVoice `model.onnx`/`tokens.txt` for final transcription and the optional streaming Zipformer encoder/decoder/joiner/tokens files for partial preview. Audio capture exposes only newly recorded PCM to the preview Worker every 120 ms; the completed in-memory recording still goes independently to SenseVoice for final injection.
- `worker/export_sensevoice_onnx.py` converts an already verified official SenseVoice model without changing its weights or quantizing it. `scripts/build-native-runtime.ps1` emits the existing `rain-worker/rain-worker.exe` package layout, so the Tauri Worker client requires no protocol rewrite.
- `scripts/compare-workers.py` supports one-audio comparison and deterministic AISHELL-1 corpus mode, keeps each Worker alive across the corpus, computes punctuation-insensitive CER, and applies a configurable native-versus-Python CER-gap gate. After two passing runs, new installations select the native component; legacy SenseVoice directories without both native files keep the Python path until the user explicitly downloads the native model version.
- Model files may carry an optional per-file HTTPS URL. This is used for Rain's verified ONNX/tokens Release assets while upstream ModelScope files retain the repository download route.
- Worker processes are assigned to a kill-on-close Windows Job Object. Shutdown first gives the Worker 500 ms to exit normally, then kills it if necessary. Three consecutive crashes disable automatic restart until the user explicitly checks the Worker.
- FunASR runtime dependencies include both `torch` and the matching `torchaudio`; Worker health checks report either one as missing before model use.
- The visual overlay is click-through; `overlay-cancel` is a separate small interactive window so the rest of the overlay never blocks the target application. Both are shown through Win32 `SetWindowPos` with `SWP_NOACTIVATE` and hidden through Win32 `ShowWindow(SW_HIDE)`; mixing native show with Tauri hide leaves Tauri's cached visibility stale.
- Global-shortcut callbacks are matched to the configured shortcut by parsed key identity (`Shortcut::id`), not display text. The backend canonicalizes aliases and modifier order (for example, `Ctrl+Shift+Space` becomes `shift+control+Space`).
- New configurations default to `Ctrl+Shift+Space`. Existing configurations retain their chosen shortcut unless the user changes it.
- Dynamic Escape registration/unregistration is serialized on a background channel. Calling the global-shortcut plugin from its main-thread callback deadlocks Tauri's event loop and freezes the main window and tray.
- The tray keeps its context menu on right click. A left-button release bypasses the menu and calls the shared `show_main` path to unminimize, show, and focus the main window.
- The custom title-bar controls require explicit Tauri 2 capability grants for `core:window:allow-minimize` and `core:window:allow-hide`; `core:default` exposes window inspection but not these mutations.
- The base Tauri NSIS contains no Worker, PyTorch, CUDA runtime, llama.cpp binary, or model weights. `scripts/build-runtimes.ps1` builds the native SenseVoice ZIP plus CPU/CUDA 12.8 PyInstaller ZIPs and one release manifest; `scripts/release.ps1` verifies the native model assets before building the lightweight signed installer.
- `RAIN_RUNTIME_MANIFEST_ENDPOINT` is a compile-time HTTPS component catalog. Development builds may use an explicitly configured absolute Python executable; a plain system `python` command is never considered a ready packaged runtime.
- Runtime-manifest requests have a 5-second total timeout so an unreachable GitHub endpoint cannot stall the model page. A `404 Not Found` keeps the current cached/development state; other request failures retain that state and show a localized unavailable message instead of raw transport details.
- Model-catalog requests also treat `404 Not Found` as an unpublished GitHub Release and keep the current embedded/cached catalog instead of surfacing the raw URL error.
- All builds embed `qixiaoyu27/Rain-VibeType` GitHub Releases under `releases/latest/download` as the default for application updates, model catalogs, runtime catalogs, and component ZIPs; release environment variables remain optional URL overrides.
- Ordinary settings saves are local-only. The frontend refreshes local runtime state only when the device preference or Python path changes and never fetches the runtime catalog during normal startup; remote runtime metadata is fetched only when a requested model dependency is missing. The backend reapplies autostart and rebuilds the tray only when their relevant configuration changes.
- The main frontend uses aqua-glass semantic CSS variables for coordinated light and dark themes. Radial background meshes, `panel`/`panel-strong`, and normal/hover shadow tokens create the surface hierarchy without extra markup; the left rail, paired Home status strips, test workspace, and detail panels share those tokens. Home is a three-row grid that matches the rail height: fixed status strips above and below with the test workspace absorbing remaining height. The Home test input is a real focus target for the global shortcut. The current first render initializes the light theme; the title-bar control switches themes during the session.
- Disabled buttons use the standard unavailable cursor; the wait cursor is reserved for actual in-progress work.
- Settings navigation is task-oriented: Home, Recording & hotkey, Text insertion, Status feedback, Model library, Text cleanup, Inference settings, App settings, and Privacy & diagnostics are separate pages. Navigation resets the content scroll position, and model entries use native `details` elements so the selected or downloading model opens while the rest remain compact summaries.
- Every non-Home settings page uses the same compact title-plus-one-line-description header; descriptions are localized in Simplified Chinese and English and truncate rather than displacing the right-side action buttons.
- `src/assets/rain.svg` is the canonical brand mark used by the custom title bar. Windows PNG/ICO assets under `src-tauri/icons/` are generated from that SVG so the UI, executable, taskbar, Start menu, and tray share one cloud-rain-audio identity.
- `docs/MACOS_MIGRATION.md` is the Apple Silicon handoff guide: shared Tauri/Worker layers stay reusable while `platform_windows.rs`, Windows-only dependencies, runtime artifacts, and NSIS packaging must be replaced behind platform gates.
