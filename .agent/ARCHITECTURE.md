# Architecture

- `src-tauri/src/main.rs` owns Tauri commands, tray/hotkey behavior, onboarding, updater integration, model lifecycle, and the `Idle → Recording → WaitingForModel → Transcribing → Injecting` state machine.
- `src-tauri/src/audio.rs` captures microphone samples only in memory, mixes channels to mono, resamples to 16 kHz PCM 16-bit, tracks level, and surfaces device errors.
- Optional playback ducking uses the Windows Core Audio default render endpoint. A recording-scoped guard stores the original master-volume scalar, sets it to 20% while capturing, and restores it on stop, cancellation, or task failure.
- `src-tauri/src/platform_windows.rs` captures/revalidates foreground process identity; owns per-format clipboard snapshot/restore, Unicode input, work-area/fullscreen queries, sounds, and the Worker Job Object.
- Windows target capture runs before runtime/model checks and only requires a valid foreground window/process. Completion validation uses the stable foreground process ID because Chromium/Electron may omit `hwndFocus` or recreate internal/top-level HWNDs. Text goes to the current cursor in the originally captured application; changing processes falls back to clipboard.
- Clipboard paste is the default output path. Rain deep-copies movable clipboard formats and enhanced metafiles, restores them after a 300 ms paste window only when the sequence still belongs to Rain, and preserves concurrent user copies. GDI bitmap handles are skipped because their `CF_DIB` representation carries the bitmap data.
- `src-tauri/src/models.rs` is the manifest-driven repository layer. It handles resumable official downloads, per-file SHA-256 checks, atomic promotion, directory/ZIP import, custom-model marking, verification, independent HTTPS catalog updates, retained old versions, and root-confined deletion.
- `src-tauri/src/runtimes.rs` manages separately published CPU/NVIDIA PyInstaller Workers. It detects NVIDIA once per process through a hidden `nvidia-smi`, resolves `auto|cuda|cpu`, downloads only after explicit user action, resumes a component ZIP, checks size and SHA-256, rejects unsafe ZIP paths/symlinks, and atomically installs below the application data `runtimes` root.
- Every ModelScope HTTP client must send the versioned `RainVibetype/<version>` User-Agent. ModelScope's LFS CDN rejects reqwest's missing/default identity with `403 denied by UA ACL`, even for public models.
- `src-tauri/resources/models.json` is the versioned source of model metadata, official revisions, required files, sizes, and hashes.
- `src-tauri/src/worker.rs` owns one long-lived Worker process, JSON control frames plus binary PCM, request IDs, response timeout, loaded-model caching, unload, crash fuse, and bundled/development executable selection.
- `worker/rain_worker.py` provides one adapter interface for SenseVoice, Fun-ASR Nano, and Paraformer-zh. It loads local paths only and supports CPU/CUDA selection, cancellation semantics, and normalized errors.
- `native-worker/` implements the same line-delimited JSON plus binary PCM contract in Rust with sherpa-onnx. The first vertical slice supports unquantized SenseVoice on CPU and expects `model.onnx` plus `tokens.txt` beside the managed FunASR model files.
- `worker/export_sensevoice_onnx.py` converts an already verified official SenseVoice model without changing its weights or quantizing it. `scripts/build-native-runtime.ps1` emits the existing `rain-worker/rain-worker.exe` package layout, so the Tauri Worker client requires no protocol rewrite.
- The native runtime remains behind a parity gate. `scripts/compare-workers.py` feeds identical decoded PCM to the native and Python Workers and reports text/load/inference differences; the production runtime manifest must not select the native component until corpus validation passes and ONNX files have an official release source.
- Worker processes are assigned to a kill-on-close Windows Job Object. Shutdown first gives the Worker 500 ms to exit normally, then kills it if necessary. Three consecutive crashes disable automatic restart until the user explicitly checks the Worker.
- FunASR runtime dependencies include both `torch` and the matching `torchaudio`; Worker health checks report either one as missing before model use.
- The visual overlay is click-through; `overlay-cancel` is a separate small interactive window so the rest of the overlay never blocks the target application. Both are shown through Win32 `SetWindowPos` with `SWP_NOACTIVATE` and hidden through Win32 `ShowWindow(SW_HIDE)`; mixing native show with Tauri hide leaves Tauri's cached visibility stale.
- Global-shortcut callbacks are matched to the configured shortcut by parsed key identity (`Shortcut::id`), not display text. The backend canonicalizes aliases and modifier order (for example, `Ctrl+Shift+Space` becomes `shift+control+Space`).
- New configurations default to `Ctrl+Shift+Space`. Existing configurations retain their chosen shortcut unless the user changes it.
- Dynamic Escape registration/unregistration is serialized on a background channel. Calling the global-shortcut plugin from its main-thread callback deadlocks Tauri's event loop and freezes the main window and tray.
- The custom title-bar controls require explicit Tauri 2 capability grants for `core:window:allow-minimize` and `core:window:allow-hide`; `core:default` exposes window inspection but not these mutations.
- The base Tauri NSIS contains no Worker, PyTorch, CUDA runtime, or model weights. `scripts/build-runtimes.ps1` builds separate CPU and CUDA 12.8 PyInstaller onedir ZIPs plus a release manifest; `scripts/release.ps1` then builds the lightweight signed installer.
- `RAIN_RUNTIME_MANIFEST_ENDPOINT` is a compile-time HTTPS component catalog. Development builds may use an explicitly configured absolute Python executable; a plain system `python` command is never considered a ready packaged runtime.
- All builds embed `qixiaoyu27/Rain-VibeType` GitHub Releases under `releases/latest/download` as the default for application updates, model catalogs, runtime catalogs, and component ZIPs; release environment variables remain optional URL overrides.
- Ordinary settings saves are local-only. The frontend refreshes runtime state only when the device preference or Python path changes; the backend reapplies autostart and rebuilds the tray only when their relevant configuration changes.
- The main frontend uses neutral semantic CSS variables for high-contrast dark and light themes. `rain-theme` in `localStorage` stores an explicit choice; otherwise the first render follows `prefers-color-scheme`, and the title-bar toggle updates the root `data-theme` before application data loads.
